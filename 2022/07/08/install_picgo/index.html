怎样用PicGo+Github搭建免费的图床

发表于2022-07-08|更新于2022-07-19|学习随笔
|字数总计:1.3k|阅读时长:4分钟|阅读量:

序

别问我为啥之前的blog没有图片，因为我没有配置好图床。发现没有图片blog就非常没有意思，而且没有说服力，所以今天用picgo和github搭建了一个免费的图床。

壹 · 下载安装PicGo

下载

PicGo官网有详细的安装指南，按照上面的指导下载安装就行了，我下载的不是最新发布的版本，是比较稳定的2.3.0-beta.8版本，根据自己电脑情况选择安装包下载即可。

安装

由于我上学期中途尝试配置图床，下载安装了picgo，后面又因为太忙（懒）了，想着还是暑假再弄吧，然后把picgo卸载了，当时是安装在E盘的，所以直接把整个丢进了回收站。今天安装的时候出现了问题。安装时提示了安装已中止，安装程序并未成功地运行完成。

排查原因：可能是因为有原先的安装记录没有删除。

首先在cmd中用where指令where picgo查找picgo的位置，一般在C:\Users\ASUJS\AppData\Local中，找到后删除原先的安装记录。或者直接用del指令del picgo删除。

友情提示：AppData文件夹一般是隐藏文件夹，可以在文件资源管理器上方的”查看“中勾选”隐藏的项目“即可找到。

但是，我删除完以后再重新安装picgo还是出现了相同的问题，上网查了好多方法都不能解决这个问题。

不能放弃！不能放弃！不能放弃！

我想到了在Windows设置中，有一个应用选项可以卸载应用。

点击进入应用，在旁边相关设置下方有程序和功能选项，点击进入，在里面找到了原来安装的picgo，右键单击卸载。

最后重新安装picgo，成功nice！

保存一下卸载或更改应用程序的路径控制面板\程序\程序和功能，以后出现类似的问题也不怕啦。

贰 · 新建GitHub仓库

新建GitHub仓库

打开Github，New a repository（新建一个仓库）存放上传的图片。

获取token

在picgo中配置的时候需要填写一个token，可以现在获取待会直接填写即可。

在GitHub中点击头像旁边的小箭头展开，点击进入Settings，在左旁栏中点击进入Developer settings，在左旁栏中点击进入Personal access tokens，点击Generate new token。Note中可以随便填写，Expiration是token的有效期限，可以设置不过期，也可以设置有限日期，但是如果设置有限日期要记得及时更新，勾选repo选项，然后点击下方的Generate token就生成了一个新的token。

生成新的token以后一定要记得保存，因为这个token只会出现一次，点击旁边的copy按钮可以直接复制。

叁 · 配置PicGo

配置GitHub图床

打开picgo，点击左边图床设置中的GitHub图床，进入设置界面。

  • 设定仓库名，填写刚刚在GitHub中新建的仓库名：Owner/Repository name
  • 设定分支名，填写main
  • 设定Token，填写刚刚复制的token即可
  • 指定存储路径，会在刚刚创建的仓库中新建一个文件夹存放图片。
  • 设定自定义域名，这是在网上找到的，https://cdn.jsdelivr.net/gh/ + 仓库名可以加速。

填写完以后，点击确认即可。

验证图床是否配置成功

点击左边上传区，下面选择URL链接格式，拖一张照片到上传区，等待进度条完成。点击左边相册，可以看到是否上传成功。点击图片下方的copy按钮可以复制图片的URL，在浏览器中输入即可看到图片。如果你想上传网上的图片，可以在上传区选择快捷上传中的URL，输入图片的URL即可上传。成功啦啦啦~

肆 · PicGo+Typora写Markdown

如果有Typora的话，可以配置一下，写Markdown是直接拖入图片即可上传图片。点击文件，选择偏好设置，点击图像。上传服务服务选择PicGo(App)，PicGo路径填写PicGo.exe可执行文件所在路径，一般在安装的文件夹中，点击验证图片上传选项，验证成功。如果上传不成功，检查一下picgo设置里的设置Server，监听端口为36677。

如果没有Typora，可以用vscode写Markdown，安装一个扩展Markdown Preview Enhanced，非常方便。

尾

总结：

  • 用PicGo+Github搭建图床总体还是比较简单的，有大佬直接用PicGo-Core和cmd操作，占用内存小一些，有兴趣朋友可以看看。我感觉还是用PicGo(App)比较方便。
  • Blog第一次写这么多字，中途不小心关掉了，还好我在开始的时候另存了一份文档（吸取之前写作文没有保存的经验了，呜呜呜……），有一种写实验报告的感觉，不过还是很有成就感滴！
  • 最后还是要鸡汤一下，有错误和bug很正常，要有一颗never give up的心！
