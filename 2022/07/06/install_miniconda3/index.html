怎样在Windows10下用Miniconda3+PyCharm搭建Tensorflow2环境

发表于2022-07-06|更新于2022-07-12|学习随笔
|字数总计:2k|阅读时长:6分钟|阅读量:

序（fei hua）

  • 因为大一的时候误打误撞地参加了srdp项目，本来想着可以跟学长学姐们学习一下，我就打个杂什么的，但是我报名的这个项目百分之九十的任务都在我身上，心情复杂。因为这个完成这个项目计划的时间是一年，由于上学期摆烂，眼看着时间已经过了一半，如果暑假不弄的话，下学期也有可能没有时间搞，所以下定决心暑假要做完，否则下学期期末肯定交不了差。所以就开始了艰难的配环境之路。
  • 电脑里原本安装了Python 3.9和PyCharm，但是指导的学长说用Python 3.7和TensorFlow 2.7.0比较稳定。本来想着用华为云的虚拟机上下载安装，但是家里网络不太稳定，所以还是用自己的电脑下载安装吧。看到网上教程一般用Anaconda+Spyder比较多。开始的时候我下载了Anaconda3，按照网上的教程安装完之后，感觉有许多用不到或者不会有的包或工具，很占内存，我尝试用PyCharm创建conda虚拟环境的时候遇到了困难，上网找了好多资料也没有解决问题，然后我就把Anaconda3删了。看到网上有介绍可以安装Miniconda3，跟Anaconda3功能差不多，都是包管理工具，而且它只包含最基本Python和conda以及相关的必须的依赖项，比较适合我这种green hand，所以经过几次捣腾之后，终于配置好了Miniconda3+PyCharm搭建Tensorflow2环境。

壹·下载安装Miniconda3

下载

可以到Miniconda官网或者镜像网站下载，根据自己的电脑选择合适的版本，我下载的是Python 3.7版本对应的Miniconda3。

  • 官网：Miniconda — Conda documentation
  • 镜像：Index of /anaconda/miniconda/ | 清华大学开源软件镜像站 | Tsinghua Open Source Mirror，可以下载Miniconda3-3.7.x对应的版本。

不禁想吐槽一下，Miniconda3才几十M，而Anaconda有几百M。

安装

如果英语好的话，可以直接看官方文档Installing on Windows — conda 4.13.0.post36+86447b6b documentation安装，上面说得非常清楚。

下载完之后，双击安装包，进入安装页面，有几个需要注意的地方。

如果没有特别需要，这里建议选择”Just Me”，如果选择”All Users”，安装过程中需要管理员权限，而且后面安装过程中有一个add PATH选项会被disabled。

一般默认装在C盘，但是我一般会把软件装在E盘，在E盘提前新建一个文件夹Miniconda3。

第一项是将Miniconda3添加到我的环境变量，第二项是将Miniconda3作为我默认的Python 3.7，如果不想手动配置环境变量的话可以勾选第一项，但是我没有勾选第一项。建议勾选第二项。选择完毕后点击Install，等待安装完成即可。

添加环境变量

由于上面没有添加环境变量，所以要手动添加环境变量。

打开高级系统设置（2种方式）

  • 右键单击此电脑，打开属性，在右旁栏选择高级系统设置，打开。
  • 或者直接点击左下方Windows图标搜索高级系统设置，打开。

点击进入环境变量。

选择系统变量中的Path，点击编辑。

添加3行参数：miniconda3的安装目录，miniconda3的安装目录\Scripts，miniconda3的安装目录\Library\bin，添加完成后点击确认。

为了确保环境变量生效，最好是要重启一下电脑。

验证

重启完电脑后，打开cmd（2种方式）

  • 键盘快捷键win+r，输入cmd，回车enter。
  • 或者直接点击左下方Windows图标搜索cmd，打开。

在cmd中输入python，如果出现anaconda的提示信息，表示安装成功，而且Python 3.7为默认的python版本。

贰·创建虚拟环境

可以直接用命令行创建虚拟环境，也可以直接用PyCharm创建虚拟环境，我尝试了一下，还是觉得是直接用PyCharm创建虚拟环境比较方便。

此处附上如何直接使用命令行创建并管理虚拟环境的链接：Anaconda介绍、安装及使用教程 - 知乎 (zhihu.com)。不管是Anaconda还是Miniconda，都是使用conda命令。

下面说说如何直接用PyCharm创建虚拟环境吧。
